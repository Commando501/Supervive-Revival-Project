--- The register-pattern smoking gun ---

Compare register values across three separate ntdll crashes:

| Test              | Rbp                | R11                | Rip
|-------------------|--------------------|--------------------|------------
| DX11              | 0x537AC9E1         | 0x95654773B3BC     | 0xF0400001
| nullrhi           | 0x537AC9E1         | 0x95654773B3BC     | 0xF0400001
| -noraytracing     | 0x537AC9E1         | 0x95654773B3BC     | 0xF0400001

**These values are IDENTICAL across three different crash sessions.**
That's impossible for organic Windows crashes — normal register values
vary run-to-run (ASLR, memory-order fluctuations, etc.). Identical
values across sessions means:

**SUPERVIVE (or an embedded anti-cheat SDK) deliberately calls
`RtlFailFast()` or an equivalent hard-abort function with a hardcoded
fail-fast code. The register values are set intentionally as an
anti-tamper signature.**

Almost certainly: SUPERVIVE's anti-cheat detects the presence of
these cmdline flags (`-DX11`, `-nullrhi`, `-vulkan`, `-noraytracing`)
as forbidden dev/tester options and terminates the process.

--- Implications ---

1. **`-noraytracing` CANNOT be used as a workaround.** SUPERVIVE's
   anti-cheat rejects it just like the renderer flags.

2. **The renderer test matrix is compromised.** DX11 and nullrhi crashes
   we saw earlier are NOT natural crashes from missing D3D12Core —
   they're the SAME anti-cheat fail-fast. The reason DX11 mode initialized
   D3D11 RHI first is that D3D11 init happens BEFORE the anti-cheat
   check fires (~24s post-Join in DX11 mode). Similarly for nullrhi.

3. **The D3D12 mode "natural" crash (RVA +0x2976FF0, in our target class)
   IS STILL REAL.** Under D3D12 mode with NO cmdline flags, the client
   reaches the FD3D12CommandContext crash organically. That identification
   stands.

4. **What we CAN'T tell** from these tests: whether the FD3D12CommandContext
   crash would happen under `-noraytracing` if the anti-cheat weren't in
   the way. To determine that, we'd need to bypass or disable the
   anti-cheat check.
