=== SESSION 40 VULKAN TEST — INCONCLUSIVE (client hangs during Vulkan init) ===

After the DX11 test showed the crash isn't D3D12-specific, tried `-vulkan`
cmdline as the second renderer-variant test. Expectation was either:
(a) Vulkan crashes at the same address → class is fully renderer-independent
(b) Vulkan runs cleanly → workaround found (unlikely since DX11 didn't work)

**Actual result: (c) — Vulkan hangs during RHI init, client never boots
past the map-load phase. No connection, no menu-load, no crash reached.**

--- Test setup ---

New batch `scratchpad/launch-sv-open-vulkan.bat` = same as `-DX11` variant
but with `-vulkan` swapped in. Same session-38 stub + session-38 code
state.

--- Test result ---

Client log ends at:

    [23:43:35:094] LogRHI: Using Forced RHI: Vulkan

No further log activity. Process still running 3.8 minutes later but only
0.2 min of CPU time (5% utilization) and 231 MB working set (normal boot
is 1-2 GB). Client is stuck in RHI init.

Sentry logged a session CrashGUID `UECC-Windows-571F879A47082B8CF18E109104BF6447`
but no corresponding CrashContext.runtime-xml was produced — the process
isn't crashing, just hung.

Stub side saw no connection activity.

--- Interpretation ---

SUPERVIVE's shipping build likely doesn't ship Vulkan shaders or Vulkan
runtime binaries. Common in AAA UE 5 games where the packer / anti-cheat
is tuned specifically to DX12 code paths. `LogRHI: Using Forced RHI: Vulkan`
just prints the requested mode; the actual Vulkan driver / shader
initialization silently blocks.

Rules out `-vulkan` as a workaround, but doesn't answer the question we
wanted: is the crash class renderer-independent?

--- What this rules IN and OUT ---

RULED OUT:
- Vulkan mode as a workaround for the SUPERVIVE Revival goal
- Vulkan as a rendering path for further tests

STILL OPEN:
- Whether the crash class is renderer-independent
- What subsystem SPECIFICALLY holds the uninit TArray

--- Renderer test matrix (final) ---

| Renderer | Result                                        | Time to crash |
|----------|-----------------------------------------------|---------------|
| D3D12    | Menu loads 25 heroes, crashes after wallet    | 64 s post-Join |
| DX11     | Menu loads 25 heroes, crashes after wallet    | 24 s post-Join |
| Vulkan   | Hangs in RHI init, never boots                | N/A           |

DX11 → D3D12 delta of 40 seconds suggests the crash-triggering subscriber
adds happen at DIFFERENT rates under different renderers. That's the only
distinguishing signal we have between the two mode results.
