HOW TO READ IT
--------------
- Column [i] is the FieldNetIndex (== N in the client error).
- Index numbering is HIERARCHICAL and cumulative from the root: per class
  level, that level's owned replicated PROPERTIES come first, then its RPC
  FUNCTIONS; then the next subclass continues from there.
- FieldNetIndex != RepIndex. RepIndex counts ONLY properties (ClassReps
  position); FieldNetIndex also counts the RPC functions interleaved at each
  level. Example: TargetViewRotation is RepIndex=12 but FieldNetIndex [14],
  because AController's 2 RPCs (ClientSetLocation/Rotation) occupy net 12-13.
- The "Invalid replicated field" error loop only handles STRUCT (custom-delta)
  properties and FUNCTIONS. Scalar properties are consumed earlier by the
  RepLayout and never appear as an N here. The struct props below are marked.

USING THIS TO INTERPRET N (when step 5 runs live)
-------------------------------------------------
SUPERVIVE's client added CPF_Net properties to base classes (session 22 shows
AActor.ServerState etc.). Extra properties on the client shift EVERY later
net index UP relative to this stock table. So when the client reads a field
header our stub wrote (using stock indices), the index lands on the WRONG
client field:
  - If N (client index) points at a client SCALAR property where our stub
    meant a struct/RPC -> "Invalid replicated field N".
  - The GAP between our stock index and the client index at the first mismatch
    == the number of extra CPF_Net properties SUPERVIVE inserted before that
    point. That count + position tells us exactly how many/which fields to
    inject in step 2 to realign.
Compare the live N against this table:
  N <= 9   : divergence is inside AActor's own reps (ServerState etc. inserted
             among/after AActor properties) -> inject on AActor.
  10 <= N <= 13 : divergence at AController level.
  14 <= N <= 16 : divergence among APlayerController's own properties.
  N >= 17  : divergence has pushed into the RPC range; almost certainly caused
             by extra PROPERTIES earlier (which shifted the whole function
             block). Injecting the missing earlier properties realigns it.
